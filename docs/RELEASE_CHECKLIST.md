# AP-Univ-Planner リリース前確認チェックリスト

このチェックリストは、GitHub Pagesへ反映する前後に確認するためのものです。
READMEやCHANGELOGの更新を行わない小規模リリースでも、最低限「必須確認」は実施してください。

## このファイルの使い方

`docs/RELEASE_CHECKLIST.md` は、リリース前後の確認手順をまとめた**テンプレート兼手順書**です。

基本運用は次のとおりです。

- このファイル自体は、空欄のテンプレートとしてリポジトリにコミットしておく。
- リリース作業時は、このファイルを見ながらローカルで確認する。
- チェックを入れた作業結果は、原則としてこのファイルにはコミットしない。
- 確認結果を残したい場合は、GitHub Release本文、Pull Request本文、Issueコメントなどに記録する。
- 毎回このファイルにチェック結果を書き込んでコミットすると、次回リリース時に前回のチェック状態が残って紛らわしいため避ける。

リリースごとの確認記録をファイルとして残したい場合のみ、必要に応じて別ファイルを作成してください。例: `docs/release_checks/2026-xx-xx_v1.1.0.md`。ただし、通常はGitHub Release本文に記録すれば十分です。

## 1. 必須確認

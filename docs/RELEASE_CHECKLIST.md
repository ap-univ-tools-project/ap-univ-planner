# AP-Univ-Planner 公開前チェックリスト

このファイルはテンプレート兼手順書です。チェック済みの状態は原則コミットせず、必要な場合はGitHub Release本文やIssueに結果を残します。

変更内容に応じて、次のどちらかだけを使用してください。

- **A. 講義データ更新**: `data.js` を中心とした小規模・定期更新
- **B. 機能リリース**: HTML/CSS/JS、重要ロジック、保存形式、UIの変更

ドキュメントだけの軽微な変更では、リンクと表示だけ確認すれば十分です。

---

## A. 講義データ更新

### 更新情報

- 更新日: `YYYY-MM-DD`
- `lastUpdated`: `YYYY-MM-DD`
- 更新した講義・年度: `____________________`
- 公式情報の出典: `____________________`

### 必須確認

- [ ] `data.js` の対象講義を更新した
- [ ] `lastUpdated` を更新した
- [ ] 曜日時限・学期・集中講義フラグ等に矛盾がない
- [ ] ローカルで対象講義がカタログに表示される
- [ ] 対象講義を登録できる
- [ ] 対象講義のカテゴリとハイライトが正しい
- [ ] `node --check script.js` が通る（`script.js`を変更していない場合は省略可）
- [ ] GitHub Pages反映後、対象講義と `Data: YYYY-MM-DD` を確認した

### Release判断

- [ ] 軽微な修正なのでReleaseを作成しない
- [ ] 履修判断に影響するためPATCH Releaseを作成する
- [ ] 年度全体の更新としてReleaseを作成する

## B. 機能リリース

### リリース情報

- リリース予定日: `YYYY-MM-DD`
- ツールバージョン: `vX.Y.Z`
- 講義データ更新日: `YYYY-MM-DD` / 変更なし
- 対象ブランチ: `main`
- 確認者: `____________________`

### 1. バージョン・ファイル

- [ ] `script.js` の `APP_VERSION` が予定バージョンになっている
- [ ] GitHub Releaseタグを同じ `vX.Y.Z` にする予定である
- [ ] 講義データ変更時は `lastUpdated` を更新した
- [ ] 保存JSON構造を変更した場合だけ `APP_SAVE_VERSION` を見直した
- [ ] `index.html`, `style.css`, `script.js`, `data.js` が存在する
- [ ] `templates/diff_template.json` と `examples/sample_plan.json` が存在する

### 2. 自動・簡易テスト

- [ ] `node --check script.js` が通る
- [ ] `node tests/highlight_logic_test.js` が通る
- [ ] 変更したJSONが構文エラーになっていない

### 3. 基本操作

- [ ] 初期化エラーがない
- [ ] 所属未選択時の操作制限が正しい
- [ ] 所属コースを選択できる
- [ ] カタログの検索・フィルタ・並び替えが動作する
- [ ] 通常講義を登録・移動・削除できる
- [ ] 登録済み講義専用ダイアログが動作する
- [ ] クイック登録が動作する
- [ ] 集中講義・特別研究・その他枠を操作できる
- [ ] 手動追加が動作する

### 4. ハイライト・計算・通知

- [ ] ハイライトOFF・全科目・所属のみが切り替わる
- [ ] 通常枠と特殊枠で単色・ストライプ表示が正しい
- [ ] 登録済み枠では講義表示が優先される
- [ ] 修了要件の単位集計が正しい
- [ ] 他専攻振替上限が反映される
- [ ] 同時限・同一講義の重複通知が出る

### 5. 保存・読込・拡張

- [ ] 保存JSONをダウンロードできる
- [ ] 保存JSONを読み込んで所属・時間割・特殊枠を復元できる
- [ ] 旧形式の保存データを読み込める
- [ ] 不正な保存JSONが拒否される
- [ ] 正常なカタログ拡張JSONを読み込める
- [ ] 不正なカタログ拡張JSONが拒否される

### 6. 表示・リンク

- [ ] PC表示が崩れていない
- [ ] sticky表示が下部説明と重ならない
- [ ] 右下エディタと各ダイアログが操作できる
- [ ] READMEと画面内の使い方が現在のUIと矛盾していない
- [ ] サンプルJSON、拡張テンプレート、Issuesのリンクが動作する

### 7. GitHub Pages反映後

- [ ] 公開URLで最新版が表示される
- [ ] ツールバージョンと `Data: YYYY-MM-DD` が正しい
- [ ] 保存・読込が本番URLで動作する
- [ ] 主要操作を1回ずつ確認した
- [ ] 問題があれば差し戻せるコミットを把握している

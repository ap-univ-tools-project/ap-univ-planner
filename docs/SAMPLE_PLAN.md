# 履修計画サンプル

この文書では、AP-Univ-Plannerに同梱している履修計画サンプルについて説明します。

## サンプルファイル

- [`examples/sample_plan.json`](../examples/sample_plan.json)

このサンプルは、**情報システム専攻の履修例**です。  
サンプルとして公開する履修例は、この1件のみです。

## 目的

このサンプルは、以下を確認するためのものです。

- 保存済み履修計画JSONの形式
- アプリへの読込手順
- 時間割・特殊枠・修了要件進捗の表示例
- 情報システム専攻として読み込んだ場合のカテゴリ表示例

このサンプルは、推奨履修モデルや修了保証ではありません。

## 使い方

1. [`examples/sample_plan.json`](../examples/sample_plan.json) をダウンロードします。
2. AP-Univ-Plannerを開きます。
3. 画面上部の **読込(.json)** を選択します。
4. ダウンロードした `sample_plan.json` を読み込みます。
5. 時間割、特殊枠、修了要件進捗、通知欄の表示を確認します。

読み込み後は、必要に応じて自分の所属・履修計画に合わせて編集してください。

## サンプルの内容

このサンプルは、アプリの **保存(.json)** から出力した形式です。

主な内容は以下です。

- `app`: `ap-univ-planner`
- `version`: `2`
- `myCourse`: `1`（情報システム専攻）
- `state`: M1前期、M1後期、M2前期、M2後期の履修計画

## 注意事項

- 実際の履修登録・修了判定は、必ず大学公式のシラバス、学生便覧、学務システム、学内掲示等で確認してください。
- サンプルの内容は、年度や公式情報の更新により実際の開講状況と合わなくなる可能性があります。
- サンプルを自分の計画の土台として使う場合は、読み込み後に必ず内容を確認・編集してください。

## 管理者向けメモ

サンプルJSONを更新する場合は、以下を確認してください。

- `myCourse` が `"1"` であること
- アプリの **読込(.json)** から正常に読み込めること
- 氏名、学籍番号、メールアドレス、研究室名、個人的なメモなどの非公開情報が含まれていないこと
- README、アプリ内リンク、この文書のファイル名・説明が一致していること

詳細な保守手順は [`docs/MAINTAINER_GUIDE.md`](MAINTAINER_GUIDE.md) を参照してください。

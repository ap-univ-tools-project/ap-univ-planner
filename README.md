# AP-Univ-Planner

愛知県立大学大学院 情報科学研究科 博士前期課程向けの、非公式の履修計画支援Webアプリです。  
M1前期からM2後期までの履修計画を時間割形式で整理し、修了要件の単位状況や重複登録を確認できます。

> [!IMPORTANT]
> 本ツールは大学公式のシステムではありません。掲載している講義情報や計算結果は目安です。  
> 履修登録・修了判定の最終確認は、必ず大学公式のシラバス、学生便覧、学務システム、学内掲示等で行ってください。

## 主な機能

- **2年間の履修計画管理**  
  M1前期、M1後期、M2前期、M2後期の4学期を時間割形式で管理できます。

- **所属専攻に基づくカテゴリ判定**  
  所属コースを基準に、共通科目、自専攻科目、関連科目、他専攻科目、演習、特別研究などを判定します。

- **修了要件の進捗確認**  
  専門科目、共通科目、専攻科目、関連科目、情報科学演習、情報科学特別研究の単位状況を確認できます。

- **他専攻科目の振替計算**  
  他専攻科目の専攻科目振替、関連科目振替を上限付きで計算します。

- **講義カタログ**  
  科目名検索、科目群フィルタ、学期・時間帯フィルタ、並び替えに対応しています。

- **クイック登録**  
  時間割の空き枠を選択すると、その枠に開講される講義や該当学期の情報科学演習を素早く登録できます。

- **特殊枠管理**  
  集中講義、特別研究、その他枠を通常の曜日時限とは別に管理できます。

- **空きコマハイライト**  
  カタログ上の講義が存在する枠を色で確認できます。通常枠、集中講義枠、その他枠に対応しています。

- **JSON保存・読込**  
  作成した履修計画をJSONファイルとして保存し、次回読み込めます。

- **カタログ拡張**  
  標準カタログにない講義や、年度ごとの開講変更をJSONファイルで追加・上書きできます。

## 利用上の重要な注意

本アプリはサーバーにデータを保存しません。  
ブラウザを閉じたり、ページを再読み込みしたりすると、入力中の履修計画は失われます。

作業後は必ず、画面上部の **保存(.json)** から計画ファイルをダウンロードしてください。  
次回利用時は **読込(.json)** から保存済みファイルを読み込んでください。

## 基本的な使い方

1. 公開URLにアクセスします。
2. 左パネルの **所属コース** を選択します。
3. 講義カタログを検索・絞り込みします。
4. カタログから講義を選択するか、時間割の空き枠からクイック登録します。
5. 右パネルの **修了要件 進捗** と **通知・アラート** を確認します。
6. 作業が終わったら **保存(.json)** で履修計画を保存します。
7. 次回は **読込(.json)** から保存済みデータを復元します。

## 所属コース設定について

所属コースは、科目カテゴリや修了要件計算に使われます。  
そのため、所属コース未設定の状態では、講義登録や時間割編集など一部の操作を制限しています。

所属コース未設定でも利用できる操作は以下です。

- 講義カタログの閲覧
- 検索・フィルタ・ソート
- 保存済みJSONの読込
- カタログ拡張JSONの読込

保存済みJSONに所属コース情報が含まれている場合、読込時に所属コースも復元されます。

## 保存・読込仕様

保存ファイルはJSON形式です。  
現在の保存形式では、履修計画本体に加えて、アプリ名、保存形式バージョン、保存日時などのメタ情報を含みます。

旧形式の保存データも、可能な範囲で互換読み込みします。  
ただし、読み込んだ講義が現在のカタログに存在しない場合は警告が表示されます。

本リポジトリでは、履修計画の作成例として **情報システム専攻の履修例を1件のみ** 公開しています。  
サンプルJSONは [`examples/sample_plan.json`](examples/sample_plan.json) に配置しています。

このサンプルは、あくまで入力形式や操作感を確認するための例です。

利用する場合は、[`examples/sample_plan.json`](examples/sample_plan.json) をダウンロードし、アプリ上部の **読込(.json)** から読み込んでください。

## カタログ拡張

標準カタログにない講義を追加したい場合や、年度ごとの開講変更を反映したい場合は、カタログ拡張JSONを利用できます。

### 拡張手順

1. [`templates/diff_template.json`](templates/diff_template.json) をダウンロードします。
2. JSONファイル内の `coreCourses` または `majorMasters` に講義情報を追記・修正します。
3. アプリ画面上部の **カタログ拡張(.json)** から読み込みます。

### データ構造

```json
{
  "coreCourses": [],
  "majorMasters": {
    "1": { "adv": [], "rel": [] },
    "2": { "adv": [], "rel": [] },
    "3": { "adv": [], "rel": [] }
  }
}
```

### 主なプロパティ

| プロパティ | 型 | 内容 | 備考 |
|---|---|---|---|
| `name` | string | 講義名 | 必須 |
| `schedule` | string | 表示用スケジュール | 必須。例: `前期月2`, `開講なし` |
| `sem` | string | 学期 | 前期: `z`、後期: `k` |
| `day` | number | 曜日 | 月: 1 〜 金: 5 |
| `period` | number | 時限 | 1 〜 6 |
| `isIntensive` | boolean | 集中講義 | 集中講義の場合 `true` |
| `isOther` | boolean | その他枠 | 共同研究等の場合 `true` |

### 編集時の注意

- `name` と `schedule` は必須です。
- `sem` を指定する場合は `z` または `k` を使用してください。
- `day` と `period` は片方だけ指定せず、曜日時限がある場合は両方指定してください。
- 集中講義や開講なしの科目は、曜日時限を指定しなくても構いません。
- 既存講義と同じ `name` を指定した場合、新しい内容で上書きされます。
- JSON構文エラー、型の誤り、範囲外の曜日時限などは読み込み時に検出されます。
